# Account Priority

이 스킬은 두 모드가 있다.

- `account-first`: 이미 관계가 있는 계정, 프로젝트 폴더에 context가 있는 경우
- `cold research`: 처음 보는 타깃

영업 실무에서는 `account-first` 가 더 중요하다. 이 모드에서는 generic company research가 주인공이 아니다.

## account-first에서 먼저 답할 것

1. 지금 딜 단계가 어디인가
2. 내부 챔피언이 누구인가
3. 다음 미팅/다음 액션이 무엇인가
4. 참가자 니즈가 어떻게 갈리는가
5. 리스크/블로커가 무엇인가
6. 공개 웹이 위 1~5를 어떻게 뒷받침하는가

## Brief ordering

`05-company-brief.md` 상단 순서:

1. 현재 딜 상태
2. 내부 챔피언 / buying center
3. 참가자별 니즈
4. 다음 액션 / next meeting questions
5. 공개 웹/보도/재무 보강
6. generic company context

## Parent-company rule

모회사 정보는 아래 경우에만 앞으로 올린다.

- 예산/결재 라인을 암시할 때
- 브랜드 범위를 설명할 때
- 법인 구조 혼동을 풀 때

그 외에는 supporting note로 내린다.

## What bad output looks like

- 이미 파일럿 일정이 잡힌 계정인데 브리프 첫 문단이 "이 회사는 어떤 회사인가"로만 시작함
- 내부 챔피언/실무 니즈보다 글로벌 회사 소개가 더 길다
- 기존 context/제안서/이메일이 있는데도 공개 웹 요약이 본문 대부분을 차지함

## What good output looks like

- "누가 왜 지금 이 교육을 하려는지"가 첫 화면에서 바로 보임
- 참가자별 실습 니즈가 분리되어 있음
- 공개 웹은 계정 전략을 뒷받침하는 최소 충분 근거로 사용됨
